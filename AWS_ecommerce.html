Análisis de Amazon E-commerce: exploración y limpieza¶

Mismo flujo que en Intro_Titanic.ipynb e Intro_Netflix.ipynb, ahora con amazon_ecommerce_1M.csv (1,000,000 filas).

Pasos:

  1. Cargar y explorar los datos.
  2. Detectar nulos, duplicados y anomalías.
  3. Limpiar lo que se encuentre.
  4. Crear columnas nuevas (feature engineering).
  5. Graficar hallazgos

7. Crear columnas nuevas (feature engineering)¶

Equivalente a lo que se hizo en Titanic/Netflix:

  • purchase_month: mes de compra (para ver tendencia en el tiempo).
  • discount_amount: cuánto dinero se ahorró en la compra (price - final_price).
  • price_tier: categoría de precio (Bajo / Medio / Alto).
  • is_high_rated: 1 si el producto tiene rating ≥ 4.5.
  • is_out_of_stock: 1 si no queda inventario.
  • is_delayed: 1 si el envío está retrasado.
In [19]:
df["purchase_date"] = pd.to_datetime(df["purchase_date"])

df["purchase_month"] = df["purchase_date"].dt.to_period("M").astype(str)

df["discount_amount"] = (df["price"] - df["final_price"]).round(2)
In [22]:
def clasificar_precio(p):
    if p < 5000:
        return "Bajo"
    elif p < 20000:
        return "Medio"
    else:
        return "Alto"

df["price_tier"] = df["price"].apply(clasificar_precio)
In [26]:
df["is_high_rated"] = (df["rating"] >= 4.5).astype(int)
In [27]:
df["is_out_of_stock"] = (df["stock"] == 0).astype(int)
In [28]:
df["is_delayed"] = (df["delivery_status"] == "Delayed").astype(int)
